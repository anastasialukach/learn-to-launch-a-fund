# Contributing to Learn to Launch a Fund

First off, thank you for considering contributing to this resource! 🎉

## How to Contribute

### Suggesting New Resources

1. **Check for duplicates**: Search existing issues and the README to ensure the resource isn't already listed
2. **Open an issue**: Use the "Resource Suggestion" template
3. **Include**:
   - Resource name and link
   - Brief description (1-2 sentences)
   - Which section it belongs to
   - Why it's valuable (optional but appreciated)

### Quality Guidelines

Resources should be:
- **Accessible**: Free or reasonably priced
- **Relevant**: Directly related to raising/operating VC funds
- **High-quality**: Well-regarded by the community
- **Current**: Published or updated within the last 5 years (exceptions for foundational content)

### What We're Looking For

- Practical guides and how-tos
- Case studies from emerging managers
- Regional resources (especially Gulf/MENA)
- Tools and platforms
- Data and benchmarks
- Books, podcasts, courses

### What We're Not Looking For

- Sales pitches or promotional content
- Outdated resources (>5 years old unless foundational)
- Paywalled content without significant value
- Duplicate resources

## Pull Request Process

1. Fork the repo
2. Create a new branch (`git checkout -b add-resource-name`)
3. Make your changes to README.md
4. Commit with a clear message (`git commit -m "Add [Resource Name] to [Section]"`)
5. Push to your fork (`git push origin add-resource-name`)
6. Open a Pull Request

### PR Guidelines

- Keep changes focused (one resource or one section at a time)
- Follow the existing markdown format
- Maintain alphabetical order within subsections where applicable
- Include a brief reason in the PR description

## Code of Conduct

Be respectful, constructive, and professional. We're all here to help emerging fund managers succeed.
